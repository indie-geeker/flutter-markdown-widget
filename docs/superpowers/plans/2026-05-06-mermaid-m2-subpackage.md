> [`2026-05-07-mermaid-m2-webview-renderer-replan.md`](./2026-05-07-mermaid-m2-webview-renderer-replan.md).

**Goal:** Ship `flutter_markdown_widget_mermaid` — a brand-new sibling pub package providing the default `MermaidRenderer` implementation backed by `flutter_js` + bundled `mermaid.js` — and publish both it and the main package to pub.dev so end users get Mermaid rendering with one `pub add`.

**Architecture:** New independent GitHub repo `flutter-markdown-widget-mermaid`, peer-cloned to the main repo. Inside, a single public class `FlutterJsMermaidRenderer` orchestrates: a private `_JsRuntime` adapter wraps flutter_js, a `MermaidJsLoader` (bundled-from-asset or user-injected), per-render theme directive injection, and SVG background post-processing. Unit tests run in the dart vm with a hand-rolled `FakeJsRuntime`; integration smoke tests exercise the real flutter_js binding on iOS/Android/macOS via GitHub Actions.

**Tech Stack:** Dart, Flutter, `flutter_js` (specific version locked in Task 0), bundled `mermaid.js@10.6.0`, `flutter_markdown_widget` (the main package, M1 interface), `flutter_test`, `integration_test`, GitHub Actions composite actions.

**Spec:** [`../specs/2026-05-06-mermaid-m2-subpackage-design.md`](../specs/2026-05-06-mermaid-m2-subpackage-design.md)

**Out of scope (future plans):**
- M3: Example app showcase + main-package README sections
- Web platform native support (deferred)
- mermaid.js 11.x upgrade
- Renderer-in-isolate

## Task ordering rationale

**M2.0 prerequisites** — Task 0: tag main pkg + version reconnaissance for `flutter_js` and `mermaid.js`.

**M2.1 core renderer** (Tasks 1–14) — Bottom-up TDD:
- Task 1 scaffolds the new repo (no behavior, just files).
- Tasks 2–6 implement leaf utilities, each with isolated tests, in dependency order: assets first, then pure utility classes (theme directive, SVG postprocess, loader, unsupported stub).
- Task 7 introduces `_JsRuntime` interface + flutter_js adapter (no test — pure delegation).
- Task 8 builds `FakeJsRuntime` so all subsequent renderer tests can run in dart vm.
- Tasks 9–12 build `FlutterJsMermaidRenderer` in slices: skeleton+init, render happy path, render errors+dispose, factories+version.
- Task 13 wires public barrel and adds boundary-enforcement test.
- Task 14 final M2.1 verification (format, analyze, full suite).

**M2.2 CI** (Tasks 15–19) — Integration smoke + GitHub Actions in dependency order: smoke fixture, then composite action, then both workflows, then verify nightly green.

**M2.3 publish** (Tasks 20–23) — Strict ordering: main pkg publishes 0.2.0 first, subpackage cuts over from path dep to `^0.2.0`, subpackage publishes 0.1.0, main pkg README links subpackage.

Each task ends with one commit (M2 repo or main repo as marked).

// Copyright 2026 The Flutter Markdown Widget Authors. All rights reserved.
// Use of this source code is governed by a BSD-style license that can be
// found in the LICENSE file.

import 'package:flutter_markdown_widget/flutter_markdown_widget.dart';

/// Injects a `%%{init: {"theme":"X"}}%%` directive at the top of a Mermaid
/// source string, mapping [MermaidTheme] to the corresponding mermaid.js
/// theme name.
abstract final class MermaidThemeDirective {
  /// Returns [source] with a theme directive prepended on its own line.
  ///
  /// If [source] already starts with a `%%{init: ...}%%` directive, it is
  /// returned unchanged (the user's explicit theme wins).
  ///
  /// Asserts when [theme] is [MermaidTheme.auto] — callers must resolve auto
  /// to a concrete theme before invoking the renderer.
  static String inject(String source, MermaidTheme theme) {
    assert(
      theme != MermaidTheme.auto,
      'auto must be resolved to a concrete theme before reaching the renderer',
    );

    if (_existingInitPattern.hasMatch(source)) {
      return source;
    }

    final mermaidName = _mapTheme(theme);
    return '%%{init: {"theme":"$mermaidName"}}%%\n$source';
  }

  static String _mapTheme(MermaidTheme theme) => switch (theme) {
    MermaidTheme.light => 'default',
    MermaidTheme.dark => 'dark',
    MermaidTheme.neutral => 'neutral',
    MermaidTheme.forest => 'forest',
    MermaidTheme.auto => 'default', // defensive fallback if assert is disabled
  };

  static final RegExp _existingInitPattern =
      RegExp(r'^\s*%%\{\s*init\s*:', multiLine: false);
}
```

import 'package:flutter_markdown_widget/flutter_markdown_widget.dart';

/// Injects a 100%-sized background `<rect>` immediately after the `<svg>`
/// open tag, using the color associated with [theme].
///
/// mermaid.js outputs SVGs with a transparent root, which makes dark-theme
/// content invisible on light surfaces (and vice versa). This pure-string
/// post-processor patches the SVG before flutter_svg ever sees it.
abstract final class MermaidSvgPostprocessor {
  /// Returns [svg] with a background rectangle injected, or the input
  /// unchanged if either:
  /// - the input is not a valid SVG (no `<svg ...>` tag found), or
  /// - a 100%-sized `<rect>` is already present (idempotent).
  static String injectBackground(String svg, MermaidTheme theme) {
    if (_alreadyHasBackground(svg)) return svg;

    final match = _svgOpenTag.firstMatch(svg);
    if (match == null) return svg;

    final color = _backgroundColor(theme);
    final insertAt = match.end;
    return '${svg.substring(0, insertAt)}'
        '<rect width="100%" height="100%" fill="$color"/>'
        '${svg.substring(insertAt)}';
  }

  static String _backgroundColor(MermaidTheme theme) => switch (theme) {
    MermaidTheme.light => '#ffffff',
    MermaidTheme.dark => '#1f2020',
    MermaidTheme.neutral => '#f8f9fa',
    MermaidTheme.forest => '#f4f4f4',
    MermaidTheme.auto => '#ffffff',
  };

  static bool _alreadyHasBackground(String svg) =>
      _existingBackgroundPattern.hasMatch(svg);

  static final RegExp _svgOpenTag = RegExp(
    r'<svg\b[^>]*>',
    caseSensitive: false,
  );

  static final RegExp _existingBackgroundPattern = RegExp(
    r'<rect[^>]*width="100%"[^>]*height="100%"',
    caseSensitive: false,
  );
}

  /// A stable identifier of the source for cache-key composition.
  /// Format: `mermaid-<version-string>`.

/// Loads `assets/mermaid.min.js` from this package's asset bundle.

/// consumers never see this class.

  ///
  /// Update this constant in lockstep with `assets/mermaid.min.js` and
  /// `assets/mermaid.min.js.sha256`.

// found in the LICENSE file.

import 'package:flutter_js/flutter_js.dart';

/// Minimal interface over a JavaScript runtime, used internally so unit
/// tests can substitute a fake without depending on `flutter_js` types.
abstract class JsRuntime {
  /// Synchronously evaluates [code] for side effects (e.g., loading
  /// mermaid.js or calling `mermaid.initialize`).
  ///
  /// Throws on parse errors or runtime exceptions raised by the script.
  void evaluate(String code);

  /// Asynchronously evaluates [code] and returns its string result.
  ///
  /// Implementations resolve when any JS Promise returned by the script
  /// settles. Throws on uncaught script-level exceptions.
  Future<String> evaluateAsync(String code);

  /// Releases native resources. Subsequent [evaluate] / [evaluateAsync]
  /// calls behave as undefined.
  void dispose();
}

/// Production adapter wrapping the real `flutter_js` runtime.
class FlutterJsRuntime implements JsRuntime {
  FlutterJsRuntime() : _runtime = getJavascriptRuntime();

  final JavascriptRuntime _runtime;

  @override
  void evaluate(String code) {
    final result = _runtime.evaluate(code);
    if (result.isError) {
      throw Exception(result.stringResult);
    }
  }

  @override
  Future<String> evaluateAsync(String code) async {
    final result = await _runtime.evaluateAsync(code);
    if (result.isError) {
      throw Exception(result.stringResult);
    }
    return result.stringResult;
  }

  @override
  void dispose() => _runtime.dispose();
}
```

Note: names are leading-capital (`JsRuntime`, `FlutterJsRuntime`) so the test file in Task 8 can implement them. They remain effectively private because the public barrel doesn't export them.

import 'dart:convert';

import 'package:flutter_markdown_widget/flutter_markdown_widget.dart';
import 'package:meta/meta.dart';

import '_js_runtime.dart';
import '_package_version.dart';
import 'mermaid_js_loader.dart';

/// Default `MermaidRenderer` implementation backed by `flutter_js`.
class FlutterJsMermaidRenderer implements MermaidRenderer {
  FlutterJsMermaidRenderer._internal({
    required MermaidJsLoader loader,
    required JsRuntime Function() runtimeFactory,
  })  : _loader = loader,
        _runtimeFactory = runtimeFactory;

  /// **Tests only.** Mirror of `_internal` exposed for unit tests so they
  /// can inject a [FakeJsRuntime].
  @visibleForTesting
  factory FlutterJsMermaidRenderer.forTesting({
    required MermaidJsLoader loader,
    required JsRuntime Function() runtimeFactory,
  }) =>
      FlutterJsMermaidRenderer._internal(
        loader: loader,
        runtimeFactory: runtimeFactory,
      );

  final MermaidJsLoader _loader;
  final JsRuntime Function() _runtimeFactory;

  JsRuntime? _runtime;
  Completer<void>? _initCompleter;
  bool _initFailed = false;
  Object? _initFailureCause;

  static const String _initJs =
      'mermaid.initialize({ startOnLoad: false, securityLevel: "strict" });';

  Future<void> _ensureInitialized() async {
    if (_initFailed) {
      throw MermaidInitializationError(
        source: '',
        cause: _initFailureCause!,
        stackTrace: StackTrace.current,
      );
    }
    if (_runtime != null) return;

    final existing = _initCompleter;
    if (existing != null) {
      // Concurrent caller — share the in-flight init.
      try {
        await existing.future;
      } catch (e, st) {
        throw MermaidInitializationError(
          source: '',
          cause: e,
          stackTrace: st,
        );
      }
      return;
    }

    final completer = Completer<void>();
    _initCompleter = completer;
    try {
      final mermaidJs = await _loader.load();
      final rt = _runtimeFactory();
      rt.evaluate(mermaidJs);
      rt.evaluate(_initJs);
      _runtime = rt;
      completer.complete();
    } catch (e, st) {
      _initFailed = true;
      _initFailureCause = e;
      completer.completeError(e, st);
      throw MermaidInitializationError(
        source: '',
        cause: e,
        stackTrace: st,
      );
    }
  }

  @override
  Future<MermaidArtifact> render(
    String source, {
    required MermaidTheme theme,
  }) async {
    await _ensureInitialized();
    // Render implementation lands in Task 10.
    throw UnimplementedError('Task 10 implements render()');
  }

  @override
  bool get isReady => _runtime != null && !_initFailed;

  .then(function(r) { return JSON.stringify({ svg: r.svg }); })
  .catch(function(e) { return JSON.stringify({ error: (e && e.message) || String(e) }); })

    } catch (e, st) {

    }

    final parsed = jsonDecode(stringResult) as Map<String, dynamic>;
    final errorMessage = parsed['error'];
    if (errorMessage is String) {

        message: errorMessage,

    final processedSvg =

    return MermaidArtifact(svg: processedSvg, intrinsicSize: intrinsicSize);
